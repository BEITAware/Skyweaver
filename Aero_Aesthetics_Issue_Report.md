# Aero Aesthetics Compliance Issue

## Overview
This issue highlights areas within the Skyweaver project where UI elements do not comply with the established "Aero aesthetics" guidelines. Specifically, the guidelines dictate that:
- Hardcoded hex colors should be avoided in favor of theme-defined dynamic resource bindings (e.g., `{DynamicResource AeroBackgroundBrush}`).
- Flat corners (`CornerRadius="0"`) should be replaced with standardized corner radii (e.g., `{DynamicResource StandardCornerRadius}`).

## Violations Found

### Flat Corners (`CornerRadius="0"`)
The following files contain flat corners which violate the Aero aesthetic guidelines:

- `./InstallationWizard/Resources/Controls/CustomContextMenuStyles.xaml` (2 occurrences)

## Recommended Actions
1. **Replace Flat Corners**: Update instances of `CornerRadius="0"` to use `{DynamicResource StandardCornerRadius}`.
2. **Replace Hardcoded Colors**: Update instances of hardcoded hex colors to use the appropriate `{DynamicResource ...}` keys defined in the theme resource dictionaries.
3. **Review Resource Dictionaries**: Ensure that base resource dictionaries (such as `AeroTheme.xaml`) themselves define these colors appropriately, and only use hardcoded colors at the very base level where themes are defined. UI controls and pages should exclusively use dynamic resources.
